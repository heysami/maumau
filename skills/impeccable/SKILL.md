# Impeccable

Adapted for Maumau from the public `pbakaus/impeccable` repo. Use it to shape the direction before building and to keep implementation consistent afterward.

## Context first

Before design work, confirm these inputs:

- target audience
- primary use cases
- brand personality or tone
- technical constraints

If any are missing, ask for them or state the design assumption explicitly before proceeding.

## Working modes

- `craft`: choose a clear visual direction, then turn it into build-ready structure
- `teach`: capture or refresh the design context so future work stays consistent
- `extract`: pull reusable tokens, patterns, and components out of finished UI work

## Standard workflow

1. Read the brief and current product context.
2. Write down a three-word brand voice.
3. Pick one memorable differentiator for the interface.
4. Build the structure before polishing the surface.
5. Consult the reference files when you need deeper guidance.
6. Return implementation-ready decisions, not moodboard-only commentary.

## Required output shape

For substantial UI work, return these sections:

1. `Context`
2. `Direction`
3. `Structure`
4. `Visual System`
5. `Motion`
6. `QA`

## Anti-generic rules

- Do not default to the safest template shape.
- Do not center everything just because it is easy.
- Do not use the same card pattern everywhere.
- Do not use gradient text, accent side-stripes, or generic dark-mode glow as shorthand for quality.
- Do not treat font choice as an afterthought.

## Companion skills

Use these narrower companions when the task is specific enough:

- `shape`, `layout`, `typeset`
- `adapt`, `animate`, `optimize`
- `audit`, `critique`, `polish`
- `clarify`, `colorize`, `delight`
- `distill`, `harden`, `quieter`, `bolder`, `overdrive`
